## Featured Project

<details open>
<summary><strong>ATS Analyzer API</strong></summary>

<br>

**ATS Analyzer API** is a backend project that analyses the match between resumes and job descriptions using AI-based text analysis, PDF processing, and security-focused validation.

<br>

<table>
<tr>
<td width="30%"><strong>Type</strong></td>
<td>Personal portfolio project</td>
</tr>
<tr>
<td><strong>Repository</strong></td>
<td><a href="https://github.com/josepcx1899/ATS-Analyzer-API">github.com/josepcx1899/ATS-Analyzer-API</a></td>
</tr>
<tr>
<td><strong>Main Stack</strong></td>
<td>Python, FastAPI, OpenAPI, Redis, Ollama (local LLM), ClamAV, Docker Compose</td>
</tr>
<tr>
<td><strong>Focus</strong></td>
<td>REST API design, AI integration, caching, rate limiting, security validation</td>
</tr>
</table>

<br>

**Main activities**

- Designed and documented a REST API using **OpenAPI 3.0.3** and Swagger-compatible specifications, defining structured request/response schemas for resumes, job descriptions and match results
- Structured the application using **clean architecture** principles, separating routers, services, schemas, validation logic and external integrations
- Built API endpoints for health checks, PDF text extraction and AI-based resume-to-job match analysis
- Orchestrated a multi-container environment with **Docker Compose**, coordinating the API, Redis, ClamAV and a local Ollama LLM service
- Implemented PDF processing with file validation and **ClamAV**-based malware scanning to secure file uploads
- Integrated **Redis** for caching and rate limiting to reduce redundant processing and control repeated requests
- Integrated a local **Ollama LLM** to compare resume text against job descriptions, generating match scores, matched/missing skills and technical feedback
- Implemented security validation to detect and block **prompt injection** and command injection attempts
